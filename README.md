# Enrutamiento estático y dinámico

## Definiciones

- **Enrutamiento estático:**  
  El administrador define manualmente las rutas en el router. Adecuado para redes pequeñas o estables, pero requiere mantener las rutas si hay cambios o fallos en la red.[11]

- **Enrutamiento dinámico:**  
  Los routers intercambian información usando protocolos y actualizan rutas automáticamente ante cambios de topología. Es ideal en redes grandes y en constante evolución, ya que reduce la intervención manual.[11]

## Diferencias clave

| Enrutamiento estático | Enrutamiento dinámico |
|----------------------|----------------------|
| Configuración manual | Configuración automática |
| Fácil de implementar | Requiere protocolos (OSPF, RIP, EIGRP) |
| Poca flexibilidad ante cambios | Se adapta a la topología cambiante |
| Bajo consumo de recursos | Mayor uso de CPU y memoria |
| Usado en redes simples | Usado en redes medianas y grandes |

## Ventajas

- **Estático:** Seguridad, sencillez y control total sobre la red.[11]
- **Dinámico:** Escalabilidad, adaptabilidad y menor mantenimiento manual.[11]

## Recomendación

> En redes pequeñas y estáticas, el enrutamiento estático suele ser adecuado.  
> En redes grandes, cambiantes o distribuidas, el dinámico es el estándar.[11]
